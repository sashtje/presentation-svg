What is SVG?
Ok, let's start. What is SVG? SVG stands for Scalable Vector Graphics. It is an XML-based markup language for describing two-dimensional based vector graphics. This means they can be searched, indexed, scripted, and compressed. Additionally, this means they can be created and edited with any text editor or with drawing software. SVG files come in two varieties. These are regular files with the extension .svg and compressed files in the format .svgz. There are also special programs that can help optimize svg files. Because SVG files, especially those exported from various editors, usually contain a lot of redundant and useless information. This can include editor metadata, comments, hidden elements, default or non-optimal values and other stuff that can be safely removed or converted without affecting the SVG rendering result.

Some facts about SVG:

  • W3C recommendation
  • Integrates with the CSS, DOM, JS and SMIL
  • Scalable
  • Don't lose any quality if they are zoomed or resized
  • Can be printed with high quality at any resolution
  • Every element and every attribute in SVG files can be animated
Some other facts about SVG. SVG has been developed by the World Wide Web Consortium since 1999. It designed specifically to work well with other web standards including CSS, DOM, JavaScript, and SMIL. This means that the display (formatting and decoration) of SVG elements can be controlled using a stylesheet. SVG can be embedded in html document directly and every element and every attribute in SVG files can be animated. The animation is implemented in SVG using the SMIL (Synchronized Multimedia Integration Language) language, also developed by the World Wide Web Consortium. You can also manipulate SVG elements with JavaScript. Or animate it using keyframes. Compared to classic bitmapped image formats such as JPEG or PNG, SVG-format vector images can be rendered at any size without loss of quality and as I say before, can be easily localized by updating the text within them, without the need of a graphical editor to do so. Additionally, it is possible to apply filters to elements of an SVG document. The use of bitmap graphics in SVG documents is widely available. You can insert elements with PNG, GIF, or JPG images. Text in an SVG graphic is text, not an image, so it can be selected and copied. And SVG is supported by all modern browsers.

Disadvantages:

  • No 3D graphics
  • Large file size
  • Difficult to use in large mapping applications
  • More details - more size of svg data
The disadvantages of SVG include: - There is no support for 3D graphics at all. - SVG inherits all the disadvantages of XML, such as a large file size. - Difficult to use in large mapping applications due to the fact that in order to correctly display a small part of the image, the document must be read in its entirety. - The more tiny details in the image, the faster the size of the SVG data grows.

Coordinates

Coordinates

Coordinate system:

For all elements, SVG uses such coordinate system. That is, the top left corner of the document is considered to be the point (0,0), or point of origin. Positions are then measured in pixels from the top left corner, with the positive x direction being to the right, and the positive y direction being to the bottom.

SVG attributes:

  • viewport
  • viewBox
  • preserveAspectRatio
To understand coordinates in svg we need to understand three of the most important SVG attributes that control coordinate systems: viewport, viewBox and preserveAspectRatio.

viewport
The viewport is the viewing area where the SVG will be visible (viewport coordinate system). The SVG viewport is similar to the viewport of the browser. A web page can be of any size; it can be wider than the viewport’s width, and is in most cases also longer than the viewport’s length. However, only portions of a web page are visible through the viewport at a time. To specify the size of the viewport we need to use the width and height attributes on the outermost svg element.

viewBox syntax:


viewBox = <min-x> <min-y> <width> <height>
Apart from the viewport coordinate system, there is also user coordinate system. This coordinate system is initially identical to the viewport coordinate system, but it can be modified with viewBox attribute. viewBox is the “real” coordinate system, it is used to draw the SVG graphics onto the canvas. The viewBox attribute takes four parameters: the min-x and min-y values determine the upper left corner of the viewbox, and the width and height determine the width and height of that viewBox.
On this slide we can see an example of viewport and viewBox together. - viewBox specifies a specific region of the canvas spanning from a top left point at (0, 0) to a point at (400, 300). - The SVG graphic is then cropped to that region. - The region is scaled up to fill the entire viewport. - The user coordinate system is mapped to the viewport coordinate system so that—in this case—one user unit is equal to two viewport units.

preserveAspectRatio syntax:


              preserveAspectRatio = defer? <align> <meetOrSlice>?
The preserveAspectRatio attribute allows you to force uniform scaling of the viewbox, while maintaining the aspect ratio, and it allows you to specify how to position the viewbox inside the viewport if you don’t want it to be centered by default. This is the official syntax for preserveAspectRatio. The defer argument is optional, and is used only when you’re applying preserveAspectRatio to an image tag. The align parameter indicates whether to force uniform scaling and, if so, the alignment method to use in case the aspect ratio of the viewBox doesn’t match the aspect ratio of the viewport. The last argument, meetOrSlice is also optional. This argument specifies whether or not the entire viewBox should be visible inside the viewport.

Basic shapes

There are several basic shapes used for most SVG drawing.

Rectangle
The rect element is used to create a rectangle. The x and y attributes define the left and top positions of the rectangle, the width and height attributes define the width and the height of the rectangle. The rx and the ry attributes rounds the corners of the rectangle, the fill property defines the fill color of the rectangle, stroke-width defines the width of the border of the rectangle and the stroke defines the color of the border of the rectangle.

Circle
To create a circle we need to use the circle element. The cx and cy attributes define the x and y coordinates of the center of the circle. If cx and cy are omitted, the circle's center is set to (0,0). The r attribute defines the radius of the circle.

Ellipse
The ellipse element is used to create an ellipse. An ellipse is closely related to a circle. The difference is that an ellipse has an x and a y radius that differs from each other, while a circle has equal x and y radius.

Line
To create a line we need to use the line element. The x1 attribute defines the start of the line on the x-axis. The y1 on the y-axis. The x2 and the y2 attributes defines the end of the line on the x and y-axis.

Polygon
The polygon element is used to create a graphic that contains at least three sides. Polygons are made of straight lines, and the shape is "closed" (all the lines connect up). The following example creates a polygon with three sides. The points attribute defines the x and y coordinates for each corner of the polygon.

Polyline
The polyline element is used to create any shape that consists of only straight lines (that is connected at several points). The points attribute defines the list of points (pairs of x and y coordinates) required to draw the polyline.

Accessibility

Accessibility. Let's see how we can ensure a great user experience for everyone with SVG pictures.

SVG as img src
For SVG in src of img tag we just need to fill alt attribute. But if we want to support users who still use older versions of Safari or iOS, we also need to add role="img".

SVG as img src
This example is fine to use as just a basic image replacement, but it doesn’t allow for us to access to contents of the SVG for either Assistive technology or CSS/JS.

Inline SVG
Inlining the SVG provides more control because the SVG source is directly available in the DOM. If SVG does not contain any visible text that describes the graphic, we need to add the alternative text (invisible) by adding inside svg tag tag title and tag description. According to the W3C specification this is enough. But unfortunately, browser support is not quite there, so we need also add id's to the title and desc tags, and also add aria-labelledby and role="img" into the svg tag.

Embed SVG via object or iframe

When we want to embed SVG via object or iframe we just need to add tabindex="0" to the object or the iframe tag, and to add a text element in the SVG that contains the content of title and possibly desc tag (for NVDA) and which is hidden.

Icons

As for accessibility for Icons, icons can be divided into several classes.

Standalone Meaningful Icon
Standalone Meaningful Icon, these icons need alternative text. We need to add title text and role attribute to the svg tag.

Standalone Decorative Icon
Standalone Decorative Icon, these icons do not need alternative text and they should be hidden from the screen reader.

Linked Icon, no text
For linked icons that are not paired with text, we can use aria-label on the a element to provide descriptive, alternative text.

Animations

Animations. There are three different ways to animate SVG.

Animating with CSS @keyframes



            

.left-leg {
  fill: orange;
  animation: dance 2s infinite alternate;
}

@keyframes dance {
  100% {
    transform: rotate(3deg);
  }
}
SVG elements can be targeted and styled with CSS. Meaning, you can apply animation through @keyframes. Like in the example on the slide. This kind of animation is good for simple animations when we want to animate properties that CSS can animate.

Animating with SMIL
Synchronized Multimedia Integration Language or SMIL is a syntax for animations built right into SVG. It is more powerful and can animate more things including the shape of the elements themselves.

Animating with JavaScript

  • Snap.svg
  • SVG.js
  • D3.js
  • GreenSock
  • and others ...
JavaScript can do everything what css and SMIL animations can do. So we can write animations by ourselves or use animation frameworks that work with SVG. These for example, Snap.js, SVG.js, D3.js, GreenSock and others.
